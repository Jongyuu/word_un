## 1. 产品定位

### 1.1 产品本质

Word Universe **不是**背单词工具，**是**一个知识探索游乐场。

### 1.2 核心差异化

| 传统背单词APP | Word Universe |
|------------|---------------|
| 单词列表 + 搜索框 | 知识星系图谱 |
| 记忆驱动 | 好奇心驱动 |
| 学习感强 | 游戏感强 |
| 任务式 | 探索式 |
| 线性进度 | 无限网络 |

### 1.3 一句话定位

**"在知识宇宙中漫游，让英语学习像逛维基百科一样上瘾"**

### 1.4 核心理念

- 学习来自探索，而不是记忆
- 用户进入产品后，不应该面对单词列表和搜索框
- 应该像知识图谱、维基百科网络、Obsidian Graph View一样，通过点击节点不断探索

## 2. MVP 范围

### 2.1 核心功能（必须有）

✅ **知识图谱展示**
   - 中心节点 + 周围关联节点
   - 支持拖拽、缩放、平移

✅ **节点探索交互**
   - 点击节点 → 成为新中心
   - 展开新的关联节点
   - 平滑过渡动画

✅ **节点信息卡片**
   - 单词/概念释义
   - 关系类型标签
   - 最小化设计（不打断探索）

✅ **初始数据集**
   - 100个核心单词节点
   - 5种基础关系类型（同义、反义、词根、分类、场景）

### 2.2 不在 MVP 范围（后续迭代）

- ❌ 用户登录/注册
- ❌ 学习进度跟踪
- ❌ 搜索功能
- ❌ 收藏/笔记
- ❌ 社交分享
- ❌ 个性化推荐
- ❌ 发音/例句

### 2.3 MVP 验证目标

**核心指标：用户愿意持续点击 > 5 分钟**

## 3. 核心交互流程

### 3.1 主流程

```
用户进入 
  ↓
【宇宙视图】显示初始星系
  ↓
点击任意节点
  ↓
【动画过渡】节点移动到中心
  ↓
【展开关联】周围出现新的关联节点
  ↓
显示节点信息卡片（3秒后自动收起）
  ↓
继续点击探索 → 循环
```

### 3.2 交互细节

#### 初始状态
- 屏幕中心：1个"起点"节点（如 "universe"）
- 周围：8-12个一级关联节点
- 节点大小反映重要性/连接数

#### 点击节点后
- 被点击节点动画移动到中心（0.6s）
- 旧的关联节点淡出（0.3s）
- 新的关联节点从中心向外展开（0.8s）
- 右侧滑出信息卡片（可手动关闭）

#### 自由探索
- 支持拖拽画布
- 滚轮缩放
- 历史路径面包屑（左上角）
- 返回上一个节点（左上角）

## 4. 数据结构设计

### 4.1 节点数据模型

```typescript
interface Node {
  id: string;              // 唯一标识
  type: NodeType;          // 节点类型
  label: string;           // 显示名称（英文）
  labelCn?: string;        // 中文名称
  definition: string;      // 释义
  importance: number;      // 重要性权重（影响节点大小）
  metadata?: {
    etymology?: string;    // 词源
    category?: string;     // 分类
    tags?: string[];       // 标签
  }
}

enum NodeType {
  WORD = 'word',           // 单词
  TOPIC = 'topic',         // 主题
  ROOT = 'root',           // 词根
  CONCEPT = 'concept',     // 概念
  PERSON = 'person',       // 人物
  PLACE = 'place',         // 地点
  MYTH = 'myth'            // 神话
}
```

### 4.2 关系数据模型

```typescript
interface Relation {
  id: string;
  source: string;          // 源节点ID
  target: string;          // 目标节点ID
  type: RelationType;      // 关系类型
  strength: number;        // 关系强度（0-1，影响显示优先级）
}

enum RelationType {
  SYNONYM = 'synonym',         // 同义
  ANTONYM = 'antonym',         // 反义
  DERIVED_FROM = 'derived',    // 派生
  CONTAINS = 'contains',       // 包含
  RELATED_TO = 'related',      // 相关
  USED_IN = 'used_in'          // 用于场景
}
```

### 4.3 图谱状态模型

```typescript
interface GraphState {
  centerNode: Node;               // 当前中心节点
  visibleNodes: Node[];           // 当前可见节点
  visibleEdges: Relation[];       // 当前可见关系
  history: string[];              // 历史路径（节点ID）
  viewport: {
    zoom: number;
    x: number;
    y: number;
  }
}
```

## 5. 页面结构设计

### 5.1 单页面布局

```
┌─────────────────────────────────────┐
│ [← 返回] [路径面包屑...]    [⚙️设置] │  ← 顶部导航栏
├─────────────────────────────────────┤
│                                     │
│                                     │
│          【知识图谱画布】            │
│         （占满整个屏幕）             │
│                                     │
│                                     │
│                                     │
└─────────────────────────────────────┘
            ↑ 右侧滑出信息卡片
```

### 5.2 信息卡片（右侧抽屉）

```
┌──────────────────┐
│ planet           │  ← 单词/节点名
│ /ˈplænɪt/       │  ← 发音（可选）
├──────────────────┤
│ 行星，星球         │  ← 中文释义
│                  │
│ A celestial body │  ← 英文定义
│ moving in orbit  │
│ around a star    │
├──────────────────┤
│ 🔗 词根: plan    │  ← 关系标签
│ 📦 类别: 天文     │
│ 🌍 场景: 科学     │
└──────────────────┘
```

### 5.3 节点视觉设计

- **中心节点**：最大，高亮边框
- **关联节点**：根据重要性调整大小
- **节点颜色**：按类型区分
  - 单词：蓝色系
  - 主题：紫色系
  - 词根：橙色系
  - 概念：绿色系
- **连线**：按关系类型区分颜色和虚实

## 6. 技术架构设计

### 6.1 技术栈约束

- ✅ React + TypeScript
- ✅ 优先Web，后续可打包APP
- ✅ 免费部署
- ✅ 单人开发友好
- ✅ AI辅助生成数据

### 6.2 架构分层

```
┌─────────────────────────────────┐
│   UI Layer (React Components)   │  ← 画布、节点、卡片
├─────────────────────────────────┤
│   State Management (Zustand?)   │  ← 图谱状态
├─────────────────────────────────┤
│   Graph Engine (React Flow?)    │  ← 图谱渲染引擎
├─────────────────────────────────┤
│   Data Layer (JSON / IndexedDB) │  ← 本地数据存储
└─────────────────────────────────┘
```

### 6.3 数据策略（MVP）

- **静态JSON文件**：初始100个节点 + 关系数据
- **懒加载**：仅加载当前中心节点的关联数据
- **无后端**：纯前端应用
- **AI生成**：使用 Claude 批量生成节点关系JSON

## 8. MVP 成功指标

### 8.1 技术指标
- 图谱加载 < 2s
- 节点切换动画 < 1s
- 支持 > 100 节点无卡顿

### 8.2 体验指标
- 用户平均探索时长 > 5分钟
- 单次会话点击节点 > 10次
- 用户主观反馈："我忘了这是在学英语"

## 9. 开发注意事项

### 9.1 遵循增量开发原则
- ⚠️ 禁止一次性生成整个项目
- ✅ 每个阶段完成后等待验证
- ✅ 优先实现核心交互，再补充UI细节

### 9.2 MVP 验证优先
- 先做出能点击探索的原型
- 再优化动画和视觉
- 最后扩展数据量

### 9.3 测试方法
- **功能测试**：每个阶段完成后手动测试核心交互
- **性能测试**：使用 Chrome DevTools 监控帧率
- **用户测试**：邀请真实用户体验并记录探索时长
