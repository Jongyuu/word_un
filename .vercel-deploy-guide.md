# Vercel 自动部署指南

## 📋 部署步骤

### 1. 访问 Vercel
前往 [vercel.com](https://vercel.com) 并登录（使用 GitHub 账号登录）

### 2. 导入项目
1. 点击 "Add New..." → "Project"
2. 从 GitHub 仓库列表中选择 `word_un`
3. 点击 "Import"

### 3. 配置项目
**Framework Preset**: Vite  
**Root Directory**: ./  
**Build Command**: `npm run build`  
**Output Directory**: `dist`  
**Install Command**: `npm install`

### 4. 配置环境变量
在 "Environment Variables" 部分添加：

```
VITE_SUPABASE_URL=your-supabase-project-url
VITE_SUPABASE_ANON_KEY=your-supabase-anon-key
```

> 💡 从 Supabase 项目设置中获取这些值

### 5. 部署
点击 "Deploy" 按钮，Vercel 将自动：
- 安装依赖
- 运行构建
- 部署到 CDN
- 生成预览 URL

### 6. 自动部署配置
部署完成后，每次推送到 `master` 分支都会自动触发新的部署。

## 🔗 部署后的 URLs

- **Production**: `https://word-un.vercel.app`（或你的自定义域名）
- **Preview**: 每个 PR 都会生成预览 URL

## ⚙️ 高级配置

### 自定义域名
1. 在 Vercel 项目设置中找到 "Domains"
2. 添加你的域名
3. 配置 DNS 记录（Vercel 会提供具体指引）

### 环境变量管理
- **Development**: 用于本地预览
- **Preview**: 用于 PR 预览
- **Production**: 用于生产环境

建议为不同环境使用不同的 Supabase 项目。

### 部署钩子
在 `vercel.json` 中已配置：
- API 函数内存: 1024 MB
- 最大执行时间: 10 秒

## 🚀 性能优化建议

### 1. 代码分割
当前 bundle 大小: 490.66 KB (gzip: 158.01 KB)

可以优化的点：
- 使用动态导入 `React.lazy()`
- 按路由拆分代码
- Tree-shaking 未使用的库

### 2. 图片优化
- 使用 WebP 格式
- 添加图片懒加载
- 使用 Vercel Image Optimization

### 3. 缓存策略
在 `vercel.json` 中配置缓存头：
```json
{
  "headers": [
    {
      "source": "/assets/(.*)",
      "headers": [
        {
          "key": "Cache-Control",
          "value": "public, max-age=31536000, immutable"
        }
      ]
    }
  ]
}
```

## 📊 监控和分析

### Vercel Analytics
在项目设置中启用 Analytics 可以获得：
- 页面访问统计
- Core Web Vitals
- 用户地理分布

### 性能监控
使用 Vercel Speed Insights：
```bash
npm install @vercel/speed-insights
```

然后在 `App.tsx` 中添加：
```tsx
import { SpeedInsights } from '@vercel/speed-insights/react'

function App() {
  return (
    <>
      <YourApp />
      <SpeedInsights />
    </>
  )
}
```

## 🔧 故障排查

### 构建失败
- 检查 Node.js 版本（需要 18+）
- 确保所有依赖都在 `package.json` 中
- 查看构建日志找出错误

### 环境变量问题
- 确认变量名拼写正确
- 重新部署以应用新的环境变量
- 检查变量是否在正确的环境中设置

### API 路由 404
- 确认 API 文件在 `api/` 目录中
- 检查 `vercel.json` 中的路由配置
- 查看函数日志
