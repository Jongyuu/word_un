## 1. 项目概览

### 1.1 基本信息

| 项目 | 信息 |
|------|------|
| **项目名称** | Word Universe（单词宇宙） |
| **项目定位** | 知识探索宇宙 - 在探索中自然学习英语 |
| **核心理念** | 不是背单词工具，是知识探索游乐场 |
| **当前版本** | v0.1.0 |
| **开源协议** | MIT License |
| **作者** | Jongyuu |

### 1.2 仓库信息

| 项目 | 地址 |
|------|------|
| **GitHub 仓库** | https://github.com/Jongyuu/word_un |
| **Issues** | https://github.com/Jongyuu/word_un/issues |
| **在线演示** | 🚧 待部署（计划使用 Vercel） |

### 1.3 项目描述

Word Universe 不是传统的背单词工具，而是一个知识探索游乐场。通过点击节点探索知识图谱，在好奇心驱动下自然接触和学习英语单词。用户进入产品后，不应该面对单词列表和搜索框，而是像维基百科网络一样，通过点击节点不断探索，让学习像探索宇宙一样充满乐趣。

## 2. 核心功能

### 2.1 功能概览

| 功能 | 状态 | 说明 |
|------|------|------|
| **知识图谱可视化** | ✅ 完成 | 使用 React Flow 引擎，展示节点关系网络 |
| **交互式节点探索** | ✅ 完成 | 单击查看详情，双击切换中心节点 |
| **节点信息展示** | ✅ 完成 | 抽屉式信息卡片，显示释义、词源、关系 |
| **历史导航** | ✅ 完成 | 面包屑导航和返回按钮 |
| **圆形布局算法** | ✅ 完成 | 邻居节点环绕中心节点展示 |
| **拖拽缩放平移** | ✅ 完成 | 自由探索画布 |
| **按需数据加载** | ✅ 完成 | 只加载当前中心节点的邻居 |
| **数据缓存机制** | ✅ 完成 | TanStack Query 智能缓存 |

### 2.2 详细功能描述

#### 知识图谱可视化
- 中心节点 + 周围关联节点的星系布局
- 节点大小反映重要性权重（importance）
- 节点颜色按类型区分（concept, word, topic, root 等）
- 连线颜色按关系类型编码（contains, related, derived 等）
- 支持拖拽、缩放、平移交互

#### 交互式探索
- **单击节点**：显示节点信息卡片
- **双击节点**：切换为新中心，展开新的邻居节点
- **平滑动画**：节点切换和展开动画流畅自然
- **历史路径**：面包屑导航显示探索路径
- **快速返回**：返回按钮回到上一个节点

#### 节点信息卡片
- 英文单词 + 中文释义
- 英文定义
- 词源信息（etymology）
- 主题分类（category）
- 关系类型标签（视觉化展示）
- 标签云（tags）

## 3. 技术实现

### 3.1 前端架构

```
┌─────────────────────────────────────┐
│   UI Layer (React Components)       │
│   - GraphCanvas (主画布)            │
│   - CustomNode (自定义节点)         │
│   - NodeInfoCard (信息卡片)         │
│   - Breadcrumb (面包屑导航)         │
├─────────────────────────────────────┤
│   State Management                  │
│   - React Query (数据缓存)          │
│   - Zustand (全局状态)              │
│   - React Flow State (图谱状态)    │
├─────────────────────────────────────┤
│   Data Layer                        │
│   - graphApi (API 封装)             │
│   - useGraphData (数据 Hook)        │
├─────────────────────────────────────┤
│   Graph Engine                      │
│   - @xyflow/react (图谱渲染)       │
│   - 圆形布局算法                    │
└─────────────────────────────────────┘
```

### 3.2 状态管理

| 层级 | 工具 | 用途 |
|------|------|------|
| **数据缓存** | TanStack Query | API 请求缓存、加载状态管理 |
| **全局状态** | Zustand | 应用级状态（未来扩展） |
| **图谱状态** | React Flow | 节点、边、视口状态 |
| **组件状态** | React Hooks | 局部 UI 状态 |

### 3.3 数据获取流程

```
用户进入应用
  ↓
调用 /api/graph/initial
  ↓
返回中心节点 + 12个邻居
  ↓
React Query 缓存数据（5分钟）
  ↓
用户双击节点 A
  ↓
调用 /api/graph/center/A
  ↓
返回节点 A + 其邻居
  ↓
更新图谱，卸载旧节点
  ↓
用户返回时使用缓存（无需请求）
```

### 3.4 样式系统

| 技术 | 用途 |
|------|------|
| **Tailwind CSS** | 实用优先的样式框架 |
| **CSS Modules** | 组件级样式隔离（React Flow） |
| **Framer Motion** | 动画效果库（未来扩展） |
